# Гайд по онбордингу в Zephyra Technologies

## Первый день

### Утро (9:00 — 12:00)

1. Приходите в офис на улице Облачная, 42, офис 7Б. На ресепшене вас встретит Наталья Снежко или кто-то из HR-отдела.

2. Получите пропуск «Зефир-бейдж» — именную карточку с NFC-чипом. Этот бейдж открывает все двери офиса, кроме серверной комнаты «Стратосфера-мини» (доступ только для отдела облачной инфраструктуры).

3. Вам выдадут ноутбук. Стандартная конфигурация:
   - MacBook Pro M3 с 32 ГБ RAM (для разработчиков)
   - MacBook Air M2 (для не-инженерных ролей)
   - Монитор LG 27" (опционально второй по запросу через тикет в системе «Ветерок»)

4. Установите необходимое ПО через внутренний скрипт:
   ```
   curl -sSL setup.zephyra.internal | bash
   ```
   Скрипт установит: IntelliJ IDEA, Docker, ZephyrCLI, Git, Slack, и настроит VPN «Туннель».

### После обеда (13:00 — 18:00)

5. Встреча с вашим ментором. Каждому новому сотруднику назначается ментор на 3 месяца. Ментор — сотрудник с грейдом минимум Z4, из того же отдела.

6. Настройка доступов:
   - GitLab (git.zephyra.internal) — ментор добавит вас в нужные репозитории
   - Jira (tasks.zephyra.internal) — проект вашего отдела
   - Confluence (wiki.zephyra.internal) — база знаний
   - Grafana (metrics.zephyra.internal) — дашборды мониторинга
   - Slack — каналы: #general, #random, #your-team, #incidents

7. Прочитайте обязательные документы в Confluence:
   - «Кодекс Зефира» — корпоративные ценности
   - «Правила безопасности данных» — как обращаться с данными клиентов
   - «GitZephyr Flow» — процесс разработки

## Первая неделя

### День 2-3: Знакомство с продуктом

- Посмотрите запись демо Нимбус 3.0 (видео на wiki.zephyra.internal/nimbus-demo)
- Запустите Нимбус локально через Docker: `zephyr-cli nimbus start --local`
- Попробуйте подключить виртуальный датчик: `zephyr-cli sensor add --virtual --type=temperature`
- Посмотрите, как данные появляются на дашборде Grafana

### День 4-5: Первая задача

- Ментор назначит вам задачу с тегом `onboarding` в Jira
- Обычно это небольшой баг или улучшение документации
- Создайте ветку `storm/NIMBUS-{номер}`, внесите изменения, откройте PR
- Попросите ментора сделать ревью

## Первый месяц

### Обязательные мероприятия

- **«Утренний бриз»** — ежедневный стендап команды в 10:00 (15 минут)
- **«Недельный шторм»** — еженедельное демо по пятницам в 16:00
- **«Зефир-пятница»** — неформальная встреча всей компании, последняя пятница месяца. Пиццу заказывает компания

### Испытательный срок

Испытательный срок в Zephyra — 3 месяца. В конце испытательного срока проводится встреча с руководителем отдела и ментором. Критерии прохождения:
- Минимум 5 закрытых задач в Jira
- Минимум 3 одобренных PR
- Положительный отзыв ментора
- Пройден внутренний тест на знание архитектуры Нимбус (минимум 70 баллов из 100)
